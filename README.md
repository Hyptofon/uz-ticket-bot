# 🎫 UZ TicketWatcher

Telegram-бот для моніторингу квитків на [booking.uz.gov.ua](https://booking.uz.gov.ua). При знаходженні квитка — бомбардує повідомленнями за розкладом ескалації, поки квиток не зникне або ти сам не скажеш «досить».

## Як це працює

```
Квиток знайдено → 🚨 миттєво
→ 🔥 кожні 25 секунд (5 хвилин)
→ 🔔 кожну хвилину (30 хвилин)
→ 🔁 кожні 5 хвилин (нескінченно)
```

Це навмисна поведінка — дефіцитний квиток може зникнути за хвилини.

---

## Швидкий старт

### 1. Створи Telegram-бота

1. Відкрий [@BotFather](https://t.me/BotFather) в Telegram
2. Напиши `/newbot`
3. Задай ім'я та username (наприклад `MyUzWatcherBot`)
4. BotFather видасть токен — скопіюй його

### 2. Дізнайся свій Telegram Chat ID

**Варіант A:** Запусти бота локально, напиши йому `/start` — Chat ID з'явиться в консолі:
```
[START] Chat ID: 123456789 — add to ALLOWED_CHAT_IDS if needed
```

**Варіант B:** Напиши [@userinfobot](https://t.me/userinfobot) — він відповість твоїм ID.

### 3. Налаштуй конфігурацію

```bash
cp .env.example .env
```

Відкрий `.env` і заповни:
```env
TELEGRAM_BOT_TOKEN=1234567890:AAF...  # токен від BotFather
ALLOWED_CHAT_IDS=123456789            # твій chat_id
```

### 4. Запуск локально (для розробки)

```bash
npm install
npm run dev
```

### 5. Запуск в Docker (рекомендовано для 24/7)

```bash
docker-compose up -d
```

Перевірити логи:
```bash
docker-compose logs -f
```

## Конфігурація (`.env`)

| Змінна | За замовчуванням | Опис |
|--------|------------------|------|
| `TELEGRAM_BOT_TOKEN` | (обов'язково) | Токен від BotFather |
| `ALLOWED_CHAT_IDS` | (обов'язково) | Дозволені chat_id, через кому |
| `POLL_INTERVAL_SEC` | `90` | Інтервал перевірки (секунди) |
| `ALERT_ESCALATION1_INTERVAL_SEC` | `25` | Інтервал Escalation I (сек) |
| `ALERT_ESCALATION1_DURATION_SEC` | `300` | Тривалість Escalation I (сек) |
| `ALERT_ESCALATION2_INTERVAL_SEC` | `60` | Інтервал Escalation II (сек) |
| `ALERT_ESCALATION2_DURATION_SEC` | `1500` | Тривалість Escalation II (сек) |
| `ALERT_STEADY_INTERVAL_SEC` | `300` | Інтервал Steady (сек) |
| `NIGHT_THROTTLE_ENABLED` | `true` | Уповільнення вночі (тільки для пошуку) |
| `NIGHT_THROTTLE_START` | `01:00` | Початок нічного режиму |
| `NIGHT_THROTTLE_END` | `05:00` | Кінець нічного режиму |

## Розвідка API УЗ

При першому запуску рекомендується виконати probe-скрипт для підтвердження ендпоінтів:

```bash
npm run probe
```

Результати збережуться в `src/uz-client/API_NOTES.md`.

> **Примітка:** `booking.uz.gov.ua` має захист від ботів (Cloudflare, reCAPTCHA). Якщо probe не вдається — це нормально. Бот використовує cookie jar та realistic headers для роботи в звичайному режимі.
